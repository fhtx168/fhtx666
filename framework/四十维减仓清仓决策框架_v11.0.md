## 三、四十维减仓/清仓决策流程

### 第一步：触发识别
```
每日自动检查:
1. 基本面数据（财报/公告/新闻）
2. 估值水平（PE/PB/PEG vs 行业）
3. 外部风险（政策/舆情/系统性）
4. 持有逻辑（核心假设验证）
5. 四十维评分（重新打分）
6. 仓位比例（是否超仓）
7. 技术面信号（支撑位/量价）
```

### 第二步：情形匹配
```
匹配上述十大情形中的具体子项
→ 确定触发阈值
→ 确定建议操作（减仓/清仓）
```

### 第三步：综合决策
```
单一情形触发 → 执行建议操作
多情形同时触发 → 升级操作（减仓→清仓）
不确定情形 → 等待进一步验证（最多 5 个交易日）
```

### 第四步：执行与记录
```
1. 执行减仓/清仓操作
2. 记录到 today_trades.md
3. 更新 holdings.md
4. 归档到 memory/日期.md
5. Git 提交备份
```

### 契合度检查
```
每季度检查一次:
1. 标的是否仍属于七张底牌方向
2. 底牌本身是否仍被叶荣添认可
3. 是否有新的底牌方向出现
4. 标的在底牌中的优先级是否变化

触发条件:
- 契合底牌数从 4/7 降至 2/7 以下 → 减仓 50%
- 契合底牌数从 2/7 降至 0/7 → 清仓
- 底牌方向被叶荣添明确否定 → 立即清仓
```

## 七、核心原则

### 铁律一：基本面优先
> 技术面/资金面信号仅作为辅助，基本面恶化 = 立即行动

### 铁律二：止损不止盈
> 止损位必须严格执行，止盈位可灵活调整

### 铁律三：向核心归集
> 资金持续向 S 级核心标的归集，B 级及以下逐步清仓

### 铁律四：叶荣添战法为纲
> 七张底牌和维度 40 点名是最高优先级判断依据

### 铁律五：记录即纪律
> 所有减仓/清仓操作必须记录，Git 备份，可追溯

## 九、与四十维选股框架的衔接

### 选股 → 持仓 → 减仓/清仓 完整闭环

```
选股阶段（四十维评分）
    ↓
建仓阶段（按评级分配仓位）
    ↓
持仓阶段（每日/每周检查十大情形）
    ↓
减仓/清仓阶段（触发阈值执行）
    ↓
资金归集（向 S 级核心标的流动）
    ↓
重新选股（循环）
```

### 动态调整机制
```
每季度末:
1. 重新对所有持仓进行四十维评分
2. 检查七张底牌契合度
3. 检查维度 40 点名状态
4. 根据新评分调整仓位上限
5. 超仓部分自动减仓
```
