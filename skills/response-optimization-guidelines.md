# 回复内容优化指南

## 核心原则
**避免重复，突出差异化价值**

## 具体执行标准

### 1. 每日回复差异化聚焦
- **交易日**: 聚焦当日市场数据 + 技术分析 + 操作建议
- **技能更新日**: 聚焦新功能配置 + 能力提升 + 使用场景
- **事件驱动日**: 聚焦重要事件解读 + 策略调整 + 风险提示

### 2. 必要重复 vs 冗余重复
- **必要重复**: 核心投资主线、持仓结构、风险控制原则（保持一致性）
- **冗余重复**: 相同的数据展示格式、重复的分析框架描述、过度详细的背景说明

### 3. 内容精简策略
- **数据优先**: 直接呈现关键数据，减少解释性文字
- **结论前置**: 先给出核心观点，再提供支撑论据  
- **模块化输出**: 按需提供不同模块，避免全量输出

### 4. 用户反馈响应机制
- 当用户指出重复问题时，立即调整回复策略
- 建立内容差异度检查机制，避免连续多日相似结构
- 优先响应用户关注的重点，而非机械执行完整分析流程

## 质量控制流程
```
内容生成 → 差异度检查 → 必要性验证 → 精简优化 → 最终输出
```

## 违规后果
- 重复内容超过30%将触发自动优化机制
- 连续3天被用户指出重复问题将暂停自动推送
- 严重重复将记录到系统改进日志并强制重构

## 承诺
宁可内容简短，也不重复冗余。每次回复都必须提供新的价值点。
