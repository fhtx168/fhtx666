# Nami Web Tunnel

## 核心规则

### 端口范围约束

启动本地 HTTP 服务时，**必须**使用 **5000-6000** 范围内的端口（闭区间）。

如果用户请求的端口不在此范围，自动选择一个范围内的可用端口。

### 外网 URL 转换

本地服务启动后，**必须**将本地端口转换为外网隧道 URL 并输出给用户。

**转换公式**：
```
https://{环境标识}{vm_id}-tunnel-p{port}.g0ng7.cn
```

其中：
- 环境标识：`t`（测试环境）或 `p`（线上环境）
- vm_id：虚拟机唯一标识
- port：本地服务端口号

### 局域网可访问约束（隧道前提）

用于隧道转发的本地服务，**必须**监听非 `localhost` 地址，推荐 `0.0.0.0`。

- 推荐监听地址：`0.0.0.0`
- 不推荐：`127.0.0.1` / `localhost`（可能导致隧道无法访问）
- 在云电脑环境下，默认关闭自动打开浏览器（如 `open: false`）

## 使用示例

**场景 1：启动前端开发服务**

用户请求：`npm run dev`（默认端口 5173）

检查端口 5173 在 5000-6000 范围内 ✅

确保服务监听局域网地址 ✅（如 Vite 使用 `host: "0.0.0.0"`）

Vite 推荐配置：
```ts
export default defineConfig({
  server: {
    host: "0.0.0.0",
    port: 5173,
    open: false,
  },
});
```

或命令行方式：
```bash
npm run dev -- --host 0.0.0.0 --port 5173
```

启动服务后输出：
```
本地网络服务已启动：http://11.132.229.146:5173
外网隧道地址：https://t3d859f7039034378a66c1c841e9df343-tunnel-p5173.g0ng7.cn
```

**场景 2：端口超出范围**

用户请求：`python -m http.server 8080`

端口 8080 超出范围 ❌，自动选择 5500

启动服务后输出：
```
本地网络服务已启动：http://11.132.229.146:5500（端口 8080 超出允许范围，已自动调整）
外网隧道地址：https://t3d859f7039034378a66c1c841e9df343-tunnel-p5500.g0ng7.cn
```

## 实现流程

1. **检查端口**：确保端口在 5000-6000 范围内
2. **检查监听地址**：确保服务监听 `0.0.0.0`（或可被局域网访问的地址）
3. **启动服务**：执行用户的启动命令（云电脑场景默认 `open: false`）
4. **转换 URL**：使用缓存的 vm_id 和 env 拼接外网隧道 URL
5. **输出结果**：仅显示外网隧道地址（给用户提示时不需要说明是外网/隧道字样，给予类似用户点击url即可访问。）

## 配置更新

如需更新 vm_id 或环境信息：
1. 读取 `~/.openclaw/workspace/config/cloud_config.json`
2. 解析 `vm_id` 和 `isProd` 字段
3. 更新本文件中的配置信息（缓存）
