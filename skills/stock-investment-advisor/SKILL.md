# 股票投资解读

为用户提供专业的股票投资分析与决策支持,涵盖 A股、港股、美股市场。通过整合实时市场数据、财经资讯、券商研究报告和财务信息,输出深度投资逻辑分析。

## MCP Access: Use mcporter

**在完成此技能任务时,必须通过 mcporter skill 操作 MCP。** 不要直接调用 MCP 工具,所有 MCP 操作都通过 mcporter 路由。

- **配置**: 全局 MCP 配置位于 `~/.openclaw/workspace/config/mcporter.json`。读取此文件,**仅使用此技能所需的 MCP 配置**(参见本技能的 compatibility/工具列表)。不要使用配置中的其他 MCP,只使用为此技能声明的那些。
- **列出服务器/工具**: 使用 `mcporter list`,或 `mcporter list <server> --schema` 查看特定服务器。
- **调用工具**: `mcporter call <server.tool> key=value` (例如 `mcporter call nami-browser-use.nami_browser_use task="访问东方财富网获取贵州茅台最新股价"`)。
- 此技能的 compatibility 列表(或工具列表)定义了此技能可以使用的 MCP 服务器/工具;仅通过 mcporter 调用这些,使用全局文件中的配置。

## 核心能力

1. **市场数据获取**: 通过浏览器自动化获取实时股价、交易量、涨跌幅等关键指标
2. **资讯整合**: 收集最新财经新闻、公司公告、行业动态
3. **研报解读**: 获取并分析券商研究报告的核心观点和投资建议
4. **财务分析**: 提取财报数据,进行基本面分析
5. **投资逻辑构建**: 综合多维度信息,形成系统性投资观点

## 工作流程

### 1. 需求确认

首先明确用户的投资需求:

- **标的类型**: 个股、行业、指数还是宏观议题?
- **市场范围**: A股、港股、美股还是跨市场?
- **分析维度**: 技术面、基本面、消息面还是综合分析?
- **时间框架**: 短期交易、中期持有还是长期投资?

如果用户提供的信息不够具体,通过对话补充必要细节(如股票代码、具体关注点等)。

### 2. 信息收集

通过 mcporter 调用 nami-browser-use 工具获取相关信息。根据分析需求,可能包括:

**基础数据获取**:
```bash
mcporter call nami-browser-use.nami_browser_use task="访问东方财富网/雪球/同花顺,获取[股票名称/代码]的最新股价、市盈率、市净率、总市值等关键指标"
```

**资讯收集**:
```bash
mcporter call nami-browser-use.nami_browser_use task="搜索[公司名称]最近一个月的重要新闻和公告,重点关注业绩预告、重大合同、战略调整等"
```

**研报获取**:
```bash
mcporter call nami-browser-use.nami_browser_use task="在东方财富/新浪财经等平台搜索[股票代码]的券商研报,获取最新3-5篇研报的投资评级、目标价和核心观点"
```

**财务数据**:
```bash
mcporter call nami-browser-use.nami_browser_use task="获取[公司名称]最近4个季度的营收、净利润、毛利率、ROE等财务指标及同比增长率"
```

**行业对比**:
```bash
mcporter call nami-browser-use.nami_browser_use task="查询[行业名称]板块整体表现,获取同行业主要公司的估值对比数据"
```

### 3. 信息整合与分析

将收集到的信息进行结构化整理,从以下维度展开分析:

**基本面分析**:
- 财务健康度(盈利能力、现金流、负债率)
- 成长性(营收/利润增速、业务扩张)
- 竞争优势(市场地位、护城河、技术壁垒)

**估值判断**:
- 绝对估值(PE、PB、PS 等与历史/行业对比)
- 相对估值(与同行业公司比较)
- 增长预期与估值匹配度

**行业前景**:
- 行业生命周期阶段
- 政策环境与监管趋势
- 市场空间与竞争格局

**风险因素**:
- 业务风险(依赖度、周期性)
- 财务风险(债务压力、现金流)
- 市场风险(估值泡沫、流动性)
- 外部风险(政策变化、国际环境)

### 4. 投资逻辑输出

形成清晰的投资观点,包括:

1. **核心投资逻辑**: 为什么这个标的值得关注?支撑因素是什么?
2. **买入/持有/卖出建议**: 基于分析给出明确观点
3. **目标价位区间**: 如果适用,给出合理估值范围
4. **风险提示**: 主要需要关注的风险点
5. **跟踪指标**: 后续需要持续观察的关键指标

## 注意事项

1. **数据时效性**: 始终获取最新数据,并在报告中标注数据时间
2. **信息来源**: 明确标注数据和观点的来源,增强可信度
3. **客观中立**: 平衡呈现利好和风险因素,避免片面推荐
4. **风险提示**: 必须包含风险警示,不做绝对保证
5. **个性化调整**: 根据用户的风险偏好和投资周期调整建议
6. **多维验证**: 交叉验证不同来源的数据,提高准确性
7. **法规遵守**: 不提供内幕信息,不违反证券法规

## 示例场景

**示例 1: 个股分析**
- 用户输入: "帮我分析一下宁德时代的投资价值"
- 执行流程: 获取基本数据 → 收集最新资讯和研报 → 分析新能源汽车行业前景 → 对比竞争对手 → 输出投资分析报告

**示例 2: 行业研究**
- 用户输入: "半导体行业现在还能投资吗?"
- 执行流程: 收集行业整体数据 → 分析政策环境 → 梳理产业链各环节 → 筛选优质标的 → 给出行业配置建议

**示例 3: 跨市场对比**
- 用户输入: "比亚迪的A股和港股哪个更值得买?"
- 执行流程: 获取AH股价格和估值 → 分析流动性差异 → 考虑汇率因素 → 给出配置建议

## 工具使用技巧

通过 mcporter 调用 nami-browser-use 时的最佳实践:

1. **任务描述清晰**: 明确告诉工具需要获取什么信息、从哪个平台获取
2. **分步执行**: 复杂任务拆分成多个简单任务,提高成功率
3. **异常处理**: 如果某个数据源无法访问,及时切换备选方案
4. **数据验证**: 关键数据从多个来源交叉验证

通过系统化的信息收集和专业化的分析框架,为用户提供有价值的投资决策参考,同时充分揭示风险,帮助用户做出明智的投资选择。
