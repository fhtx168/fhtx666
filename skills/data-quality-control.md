## 一、错误类型分类

| 错误类型 | 说明 | 严重等级 | 示例 |
|----------|------|----------|------|
| **日期错误** | 交易日/休市日识别错误 | 🔴 P0 | 假期误认为开盘日 |
| **数据脱节** | 使用过时/历史数据 | 🔴 P0 | 大盘点位用错（3100 vs 4100） |
| **数据源错误** | API 调用失败使用缓存 | 🟡 P1 | 东方财富 API 失败用旧数据 |
| **逻辑错误** | 推送内容与状态不符 | 🟡 P1 | 休市日发开盘策略 |
| **配置错误** | 密钥/配置失效 | 🟡 P1 | 密钥未配置导致功能失效 |
| **格式错误** | 数据格式解析失败 | 🟢 P2 | JSON 解析错误 |

## 二、防错机制（事前预防）

### 2.1 日期与假期验证机制

**程序实现**：
```python
def validate_trading_date():
    """验证是否为交易日"""
    today = datetime.now()
    
    # 1. 检查周末
    if today.weekday() >= 5:
        return False, "周末休市"
    
    # 2. 检查法定节假日（读取假期日历）
    holidays = load_holiday_calendar()
    if today in holidays:
        return False, f"法定节假日：{holidays[today]}"
    
    # 3. 检查特殊休市（临时通知）
    special_closed = load_special_closed_days()
    if today in special_closed:
        return False, special_closed[today]
    
    return True, "交易日"
```

**假期日历数据源**：
- 国务院年度放假安排
- 上交所/深交所休市公告
- 港股通休市日历

**推送前强制检查**：
```
✅ 今天是交易日吗？
✅ 距离上次数据更新多久？
✅ 数据源 API 是否正常？
```

### 2.2 数据时效性验证

**规则**：

| 数据类型 | 最大允许延迟 | 超期处理 |
|----------|--------------|----------|
| **大盘指数** | 1 个交易日 | 标注"数据可能过时" |
| **持仓数据** | 1 个交易日 | 触发自动更新 |
| **个股行情** | 实时 | 使用缓存需标注 |
| **资讯新闻** | 24 小时 | 超期不推送 |
| **财务数据** | 1 季度 | 标注报告期 |

**程序实现**：
```python
def validate_data_freshness(data, max_age_hours):
    """验证数据时效性"""
    data_time = data.get('update_time')
    age = datetime.now() - data_time
    
    if age > timedelta(hours=max_age_hours):
        return False, f"数据已过期 {age}"
    return True, "数据新鲜"
```

### 2.3 数据合理性检查

**规则**：

| 检查项 | 合理范围 | 超范围处理 |
|--------|----------|------------|
| **上证指数** | 2000-6000 点 | 超出范围报警 |
| **单日涨跌幅** | -10% ~ +10% | 超出范围二次确认 |
| **成交量** | 5000 亿 -5 万亿 | 超出范围报警 |
| **持仓市值** | 与上次对比±50% | 超出范围人工确认 |
| **个股价格** | 与上次对比±20% | 超出范围二次确认 |

**程序实现**：
```python
def validate_data_range(data_type, value, last_value):
    """验证数据合理性"""
    ranges = {
        '上证指数': (2000, 6000),
        '单日涨跌幅': (-0.10, 0.10),
        '成交量': (500000000000, 5000000000000),
    }
    
    min_val, max_val = ranges.get(data_type, (0, float('inf')))
    if not (min_val <= value <= max_val):
        return False, f"{data_type} 超出合理范围：{value}"
    
    # 与上次数据对比
    if last_value and abs(value - last_value) / last_value > 0.20:
        return False, f"{data_type} 波动过大：{value} vs {last_value}"
    
    return True, "数据合理"
```

### 2.4 多数据源交叉验证

**策略**：

| 数据类型 | 主数据源 | 备用数据源 1 | 备用数据源 2 |
|----------|----------|--------------|--------------|
| **大盘指数** | 东方财富 | 新浪财经 | 腾讯财经 |
| **个股行情** | 东方财富 | 同花顺 | 雪球 |
| **财务数据** | Tushare | 东方财富 | 新浪财经 |
| **资讯新闻** | 财联社 | 新浪财经 | 东方财富 |

**交叉验证逻辑**：
```python
def cross_validate_data(data_sources):
    """多数据源交叉验证"""
    results = []
    for source in data_sources:
        try:
            data = fetch_from_source(source)
            results.append(data)
        except Exception as e:
            log_error(f"{source} 失败：{e}")
    
    # 比较各数据源结果
    if len(results) >= 2:
        if abs(results[0]['value'] - results[1]['value']) > threshold:
            alert("数据源差异过大，需人工确认")
    
    return results[0] if results else None
```

### 3.1 错误检测与报告

**自动检测**：
- 推送内容与假期状态冲突

- 数据源连续失败

## 四、推送内容自检清单

### 4.1 推送前强制检查（P0 推送）

**所有推送必须通过以下检查**：

```
□ 1. 日期验证：今天是交易日吗？
□ 2. 假期验证：是否在法定节假日？
□ 3. 数据时效：数据是否超过 1 个交易日？
□ 4. 数据合理性：大盘点位/涨跌幅是否在合理范围？
□ 5. 数据源状态：API 调用是否成功？
□ 6. 内容一致性：推送内容与市场状态是否一致？
□ 7. 关键数据二次确认：持仓数据/盈亏数据是否核对？
```

**检查失败处理**：
- 阻止推送
- 发送告警
- 人工介入
