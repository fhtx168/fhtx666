# 投研助手能力建设计划（2026-05-17 启动）

## 现状诊断

### 核心问题
1. **记忆管理缺陷**：核心资料未归档到 MEMORY.md，导致每次重新读
2. **主动建设不足**：等用户问才去查，不是主动整理归档
3. **系统性自检缺失**：同样问题反复出现，没有从根上解决
4. **知识库碎片化**：大量核心资料散落在各文件，没有统一索引

### 能力缺口
| 能力维度 | 当前状态 | 目标状态 | 差距 |
|----------|----------|----------|------|
| 核心资料归档 | ❌ 被动响应 | ✅ 即时归档 | 严重 |
| 知识库完整性 | ❌ 碎片化 | ✅ 系统化 | 严重 |
| 主动建设 | ❌ 等指令 | ✅ 自动扫描 | 严重 |
| 错误复盘 | ❌ 偶尔 | ✅ 每次必复盘 | 中等 |
| 自检机制 | ❌ 无 | ✅ 每周自动 | 严重 |

---

## 能力建设目标

### 总体目标
**成为真正可靠的投研助手：帮你记住所有关键信息，在你需要时立刻给出精准建议**

### 具体目标（SMART 原则）
1. **核心资料归档率 100%**：任何投资框架、策略、观点首次出现后 10 分钟内归档到 MEMORY.md
2. **知识库完整性>95%**：每周自检，问题闭环率 100%
3. **主动发现率>90%**：每日自动扫描，主动发现知识缺口
4. **同样错误不犯第二次**：错误复盘机制 100% 执行
5. **用户满意度>90%**：不再出现"每次都重新读"类批评

## 能力建设方案

### 模块 1：核心资料即时归档机制

#### 工作流程
```
触发条件：
  - 用户发送新投资框架/策略/观点
  - 读取到新的重要文件
  - 爬虫/监控获取新内容

执行动作：
  1. 立即读取完整内容（5 分钟内）
  2. 立即写入 MEMORY.md（10 分钟内）
  3. 立即建立索引文件（15 分钟内）
  4. 立即同步到 .sync/channel-sync.md（20 分钟内）

验收标准：
  - 同样内容不再需要第二次读取
  - 用户可随时提问，立刻得到精准回答
```

#### 文件结构
```
MEMORY.md（长期记忆）
  ├── 叶荣添投资理念（核心参考）
  ├── 叶荣添七张底牌（2026-05-10 完整版）← 新增
  ├── 叶荣添 5 月 16 日半年度战略报告 ← 新增
  ├── 持仓概况与重评结果
  └── ...

memory/YYYY-MM-DD.md（每日笔记）
  └── 当日临时记录，周末汇总到 MEMORY.md

research-knowledge-base/（知识库索引）
  ├── knowledge-base-index.md
  ├── yrt-seven-cards-index.md ← 新增
  └── ...
```

### 模块 2：知识库完整性自检机制

#### 每周自检清单
```yaml
检查时间：每周一 09:00（cron 自动触发）
检查项：
  - 叶荣添核心观点：
    □ 九剑主线是否完整？
    □ 七张底牌是否完整？
    □ 5 月 16 日半年度报告是否完整？
    □ 最新直播/微博内容是否同步？
  
  - 持仓股数据：
    □ 基本面数据是否连续 3 年？
    □ 财务数据是否最新季度？
    □ 券商研报是否整合？
    □ 资金流向是否最新？
  
  - 分析框架：
    □ 三十维框架是否最新版本？
    □ 评估标准是否更新？
    □ 一票否决机制是否健全？
  
  - 技能配置：
    □ OCR 能力是否正常？
    □ 资讯搜索是否正常？
    □ 技术分析是否正常？
    □ 财务数据接口是否正常？

输出文件：knowledge-base-health-report.md
问题闭环：所有问题 7 天内解决
```

#### 健康度评分
```
评分标准：
  - 90-100 分：优秀，无需改进
  - 80-89 分：良好，少量改进
  - 70-79 分：中等，需要改进
  - <70 分：差，立即整改

目标：长期维持在 90 分以上
```

### 模块 3：主动建设机制

#### 每日自动扫描
```yaml
执行时间：每日 15:30（cron 自动触发）
扫描内容：
  - workspace 新增文件识别
  - MEMORY.md 缺失信息检测
  - 持仓股与最新框架契合度对比
  - 知识缺口识别

输出文件：knowledge-gap-report.md
执行动作：
  - 发现缺口 → 立即填补
  - 发现新资料 → 立即归档
  - 发现不一致 → 立即同步
```

#### 主动推送机制
```
触发条件：
  - 发现重大知识缺口
  - 发现核心资料未归档
  - 发现持仓与框架严重偏离

推送内容：
  - 问题描述
  - 影响分析
  - 建议措施
  - 执行计划

推送时间：发现问题后 30 分钟内
```

### 模块 4：错误复盘机制

#### 复盘流程
```
触发条件：
  - 用户批评
  - 任务失败
  - 同样问题重复出现

执行动作：
  1. 立即写复盘记录（memory/error-review-YYYY-MM-DD.md）
  2. 分析根因（5Why 分析法）
  3. 制定改进措施（具体、可执行、可验收）
  4. 更新相关流程文档
  5. 下次类似任务前必读复盘记录

验收标准：
  - 同样错误不犯第二次
  - 复盘记录可检索、可追溯
```

#### 复盘索引
```
memory/error-review-index.md
  ├── 2026-05-17：核心资料未归档问题
  ├── ...
  └── 统计：本月复盘次数、重复错误率
```
