## 零、清理层配置（新增第 13 条规则）

### 0.1 清理规则

**自动清理以下内容：**

| 清理类型 | 判断标准 | 处理方式 |
|----------|----------|----------|
| **重复内容** | 同一观点/新闻多源重复 | 保留最早/最权威来源 |
| **过时资讯** | 超过 30 天且无长期参考价值 | 归档到历史库 |
| **无价值内容** | 纯情绪宣泄/无逻辑/无数据支撑 | 直接删除 |
| **低质内容** | 逻辑混乱/数据错误/可信度低 | 标记为低质，不纳入分析 |
| **短期噪音** | 日内波动解读/无持续性 | 仅保留摘要 |

**保留以下内容（核心干货）：**

| 保留类型 | 说明 | 保存期限 |
|----------|------|----------|
| **大盘逻辑** | 趋势研判/目标位/关键变量 | 永久 |
| **板块思路** | 主线逻辑/轮动规律/产业链 | 永久 |
| **选股观点** | 龙头识别/潜力股/目标价 | 永久 |
| **风控要点** | 风险警示/避雷案例/仓位建议 | 永久 |
| **政策解读** | 宏观政策/产业政策/长期影响 | 永久 |
| **大佬观点** | 叶荣添/但斌等核心观点 | 永久 |

### 0.3 清理脚本

```python
# skills/knowledge-base-cleanup.py
# 用途：自动清理冗余资讯，保留核心干货

def cleanup_knowledge_base():
    """清理知识库"""
    
    # 1. 重复检测
    duplicates = find_duplicates()
    for dup in duplicates:
        keep_best_source(dup)
    
    # 2. 过时归档
    old_files = find_old_files(days=30)
    for f in old_files:
        if not has_long_term_value(f):
            archive_to_history(f)
    
    # 3. 质量评估
    low_quality = assess_quality()
    for f in low_quality:
        mark_as_low_quality(f)
    
    # 4. 生成清理报告
    generate_cleanup_report()
```
