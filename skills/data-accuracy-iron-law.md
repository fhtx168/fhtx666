# 数据准确性铁律

## 核心原则
**数据求实求精准是不可违背的铁律**

## 执行标准

### 1. 数据源验证
- 必须明确标注数据来源和获取时间
- 多数据源交叉验证，单一数据源必须标注风险
- 对未来日期数据明确拒绝提供，不进行主观推演

### 2. 单位和格式校验
- 严格验证数据单位（元/千元/万元/亿元）
- 数值格式标准化，避免科学计数法歧义
- 时间格式统一为 YYYY-MM-DD

### 3. 异常值检测
- 自动识别明显异常数据（如成交额偏离正常范围）
- 对异常数据进行二次验证或明确标注
- 建立历史数据对比基准，识别不合理波动

### 4. 错误处理机制
- 发现数据错误立即停止分析并报告
- 提供修正后的准确数据，同时说明错误原因
- 建立错误案例库，防止重复犯错

### 5. 质量控制流程
```
数据获取 → 单位验证 → 范围检查 → 逻辑一致性 → 多源验证 → 分析执行
```
任一环节失败则终止流程并报告具体问题

## 违规后果
- 任何违反数据准确性铁律的行为都将被记录
- 重复违规将触发系统性改进措施
- 严重错误将导致服务暂停直至问题解决

## 承诺
宁可承认数据不可用，也绝不提供错误或未经验证的数据。
