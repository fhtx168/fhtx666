### 二、Git 同步机制

#### Git 提交流程（每小时自动）

```bash
# 自动提交脚本（cron 每小时执行）
git add .
git commit -m "Auto-sync: $(date '+%Y-%m-%d %H:%M:%S')"
git push origin main
```

#### Git 冲突处理

**优先级**:
1. 最新数据优先（时间戳最新）
2. 量化数据优先（Tushare/AKShare 为准）
3. 人工确认（两端不一致时）

**冲突解决流程**:
1. 检测冲突 → 立即告警
2. 比对时间戳 → 取最新
3. 比对数据源 → 取权威
4. 人工确认 → 两端同步

### 六、冲突解决机制

#### 数据不一致处理流程

```
1. 检测不一致 → 立即告警（两端同时通知）
2. 比对时间戳 → 取最新数据
3. 比对数据源 → 取权威数据源（Tushare/AKShare）
4. 人工确认 → 飞鸿最终确认
5. 同步更新 → 两端同时更新
6. 记录日志 → 写入 data_verification_report.md
```

#### 决策不一致处理流程

```
1. 检测决策不一致 → 立即暂停操作
2. 两端沟通 → 阐明各自逻辑
3. 量化数据对比 → 取数据支持更强的一方
4. 风险评估对比 → 取风险更可控的一方
5. 飞鸿最终决策 → 两端统一执行
6. 记录日志 → 写入决策日志
```
