# 持仓股票数据库建设说明

## 建设目标
为所有持仓股票建立完整的投资档案，包含：
- 基础信息（公司概况、主营业务）
- 投资逻辑（核心优势、与主线契合度）
- 财务基本面（业绩、毛利率、现金流等）
- 技术面分析（当前价格、持仓成本、盈亏状况）
- 风险因素（主要风险点及应对策略）
- 操作策略（持仓评价、后续建议、仓位管理）
- 协同性分析（与其他持仓的互补关系）

## 数据更新机制
1. **日常维护**：每次交易后及时更新相关股票档案
2. **定期更新**：每季度财报发布后更新财务基本面数据
3. **重大事件**：公司重大公告、行业政策变化时及时更新
4. **技术面跟踪**：每周更新价格和持仓状况

## 使用价值
1. **投资决策支持**：提供完整的个股分析框架
2. **风险控制**：明确各股票的风险点和应对策略
3. **仓位管理**：基于协同性分析优化整体配置
4. **复盘总结**：记录投资逻辑和操作效果，持续改进
