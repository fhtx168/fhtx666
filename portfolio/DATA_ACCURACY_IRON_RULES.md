# 🚨 持仓数据同步铁律（不可违反）

## 核心原则
**数据准确性 > 回复速度**
**宁可延迟回复，绝不提供错误数据**

---

## ⚠️ 历史错误记录

| 时间 | 错误 | 根因 | 影响 |

**教训**：临时修复无效，必须建立系统性防护机制

---

## 🔒 强制性数据读取流程

### 每次回复持仓相关问题前，必须执行：

```
1. 读取 portfolio/SYNC_SNAPSHOT.md
   ↓
2. 验证时间戳 ≤ 30 分钟
   ↓
3. 如过期 → 读取 portfolio/holdings.md
   ↓
4. 交叉验证关键数据（总资产、持仓数量）
   ↓
5. 在回复中注明数据来源和时间
```

### ❌ 禁止行为（零容忍）
- 使用会话缓存中的持仓数据
- 使用超过 30 分钟未更新的快照
- 不验证直接回复持仓相关问题
- 凭记忆回复持仓数量、金额等关键数据

## 🔍 自动化校验机制

### 每次持仓更新后自动执行：

1. **Git 提交**
   ```bash
   git add portfolio/holdings.md portfolio/SYNC_SNAPSHOT.md
   git commit -m "📊 持仓更新 YYYY-MM-DD HH:mm"
   ```

2. **GitHub 推送**
   ```bash
   git push origin main
   ```

3. **同步日志更新**
   - 更新 `.sync/channel-sync.md`
   - 记录最新时间戳和持仓数量

4. **微信端通知**
   - 发送同步完成消息
   - 包含最新持仓数量和时间戳

## 🚨 错误处理流程

### 发现数据不一致时：

1. **立即停止回复**持仓相关问题
2. **读取最新文件**确认正确数据
3. **向用户承认错误**并说明根因
4. **立即修复**并更新所有同步文件
5. **记录到本文件**防止再犯
