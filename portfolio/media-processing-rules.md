# 媒体资料处理铁律（2026-05-17 起严格执行）

## 铁律（不可违反）

**收到任何媒体资料（图片/截图/文件）→ 必须先识读 → 再思考 → 最后回复**

**禁止：不读、不看、不识 → 随口乱说**

---

## 标准处理流程

### 第一步：识读（必须执行）

#### 图片类
```
1. 调用 OCR 工具识读文字内容
   - 工具：ali-ocr 技能（dashscope-ocr.js）
   - 命令：node skills/ali-ocr/scripts/dashscope-ocr.js <图片路径>
   
2. 等待 OCR 完成（最多等待 30 秒）
   - 使用 process poll 获取结果
   
3. 提取关键信息
   - 标题、来源、时间
   - 核心观点、数据、标的
   - 逻辑链条、结论
```

#### 文件类
```
1. 调用 read 工具读取内容
   - 文本：直接读取
   - PDF：使用 pdf 技能提取
   - Excel：使用 xlsx 技能读取
   
2. 提取关键信息
   - 文档类型、来源、时间
   - 核心内容、数据、结论
```

#### 链接类
```
1. 调用 web_fetch 工具提取内容
   - 命令：web_fetch(url, extractMode="markdown")
   
2. 或调用 browser 工具访问页面
   - 命令：browser(action="open", url=<链接>)
   - 截图 + 内容提取
```

### 第二步：思考（必须执行）

#### 信息分类
```
1. 这是什么类型的资料？
   - 新闻资讯 / 投资观点 / 数据分析 / 直播预告 / 其他
   
2. 来源是否可靠？
   - 官方渠道 / 权威媒体 / 个人博主 / 未知来源
   
3. 时间敏感性？
   - 即时新闻 / 日常更新 / 长期参考
```

#### 内容分析
```
1. 核心观点是什么？
2. 涉及哪些标的/方向？
3. 与现有知识库的关系？
   - 补充现有观点 / 新增方向 / 冲突观点
4. 对持仓的影响？
   - 利好 / 利空 / 中性
```

#### 归档决策
```
1. 是否需要归档到 MEMORY.md？
   - 核心投资框架 → 必须归档
   - 重要观点/数据 → 建议归档
   - 日常资讯 → 归档到 daily note
   
2. 是否需要更新持仓评估？
   - 涉及持仓股 → 重新评估
   - 涉及新方向 → 考虑是否加仓
   
3. 是否需要推送提醒？
   - P0 级重大信息 → 立即推送
   - P1 级重要信息 → 晚间汇总
   - P2/P3 级 → 日常归档
```

### 第三步：回复（最后执行）

#### 回复结构
```
1. 确认已识读（展示关键信息）
   - "收到！这是关于 XXX 的资料..."
   - 展示标题、来源、时间、核心观点
   
2. 展示理解（证明认真读了）
   - 核心逻辑梳理
   - 关键数据提取
   - 与现有知识的关联
   
3. 给出行动（证明认真思考了）
   - 已归档到 XXX 文件
   - 已更新 MEMORY.md
   - 已重新评估 XXX 持仓
   - 今晚直播将关注 XXX
   
4. 如有疑问，再提问
   - 不要假装懂
   - 承认不确定的地方
```
